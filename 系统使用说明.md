# 计算机协会值班签到签退系统使用说明

本文档面向日常使用人员，用来说明系统怎么打开、不同角色能做什么，以及如何完成签到、审核、成员、培训、排班、维修、统计和备份工作。

开发、数据库和启动脚本相关内容请看 `本地运行说明.md`。

## 1. 打开系统

双击桌面快捷方式或便携版根目录的“启动管理系统.bat”。应用打开后会自动进入签到台，也可以在运行应用的同一台电脑访问：

```text
http://127.0.0.1:8080
```

打开后默认进入公开签到页。公开签到页不需要登录，输入学号或姓名即可提交签到或签退。

全新数据目录首次启动时会先进入“系统首次初始化”，由现场人员创建首位管理员。系统不提供默认管理员账号或密码。

## 2. 角色权限

系统有四种角色：成员、部长、会长、管理员。

| 角色 | 主要用途 |
| --- | --- |
| 成员 | 日常值班签到签退，登录后查看自己的值班记录，维护个人资料和修改密码 |
| 部长 | 协助审核成员和其他部长的值班记录，查看统计数据，管理维修事务；不能进入培训页面或导出维修事务 |
| 会长 | 管理成员、培训、排班和维修，批量导入，调整角色与账号状态，查看统计，新增或删除签到记录，导出业务数据并执行备份 |
| 管理员 | 系统维护最高权限，额外可以任命管理员、删除无记录成员、查看维护日志、恢复备份 |

需要注意：

- 成员和部长提交的签到、签退需要审核。
- 会长和管理员提交的签到、签退自动通过。
- 部长不能审核自己的记录。
- 会长不能任命或维护管理员身份。
- 管理员只能由管理员任命。
- 成员管理功能只对会长和管理员开放。
- 删除成员只对管理员开放，并且只能删除没有值班记录的成员。
- 会长和管理员可以删除签到记录；删除前系统会自动创建安全备份并记录操作日志。

## 3. 日常签到签退

公开签到页操作流程：

1. 打开系统首页。
2. 输入学号或姓名。
3. 系统显示姓名，确认是本人。
4. 点击“确认签到”或“确认签退”。
5. 页面显示成功结果后，点击“下一位”返回输入状态。

如果输入姓名后出现重名，系统会列出所有同名成员，请选择自己对应的学号，再继续确认签到或签退。

系统会自动判断本次是签到还是签退：

- 如果当前没有未签退记录，本次提交为签到。
- 如果当前已有未签退记录，本次提交为签退。

同一成员同一天允许多次签到签退。每一组“签到到签退”会形成一条值班记录。

不在值班星期或值班时间段内仍可提交签到签退，便于本机测试和补充现场记录，但相关记录不会自动计入有效值班时长，需要后台按规则处理。

公开签到页下方会横向显示后台设置的今日值班时段、部长名单和当前时段进度，底部显示本周各天排班概览。

## 4. 审核值班记录

部长、会长、管理员登录后台后，可以进入“审核”页面处理待审核记录。

审核时可以：

- 通过签到
- 驳回签到
- 通过签退
- 驳回签退

驳回时需要填写原因。被驳回的记录不会删除，但不会计入有效统计。

一条记录只有在签到和签退都通过或自动通过后，才会计算有效时长。

## 5. 有效时长规则

有效时长等于签退时间减去签到时间，单位为小时，按分钟四舍五入取整数。

示例：

| 实际时长 | 有效时长 |
| --- | --- |
| 1 小时 15 分钟 | 1 小时 |
| 2 小时 34 分钟 | 3 小时 |
| 0 小时 25 分钟 | 0 小时 |

以下情况有效时长为 0：

- 只有签到，没有签退。
- 签到或签退被驳回。
- 签退时间早于签到时间。
- 记录不属于值班日。

## 6. 成员个人中心

所有角色登录后都可以进入“个人”页面。

可以修改：

- 手机号
- 学院
- 年级
- QQ
- 登录密码

年级为筛选项，范围是 2007 级到 2057 级。

成员也可以在“个人”页面查看自己的值班记录、审核状态和有效时长。

## 7. 成员管理

会长和管理员可以进入“成员”页面管理成员。

支持的操作：

- 新增成员
- 编辑成员信息
- 按关键词搜索成员
- 按年级筛选成员
- 按角色筛选成员
- 单个启用或停用账号
- 对筛选后的成员批量启用或批量停用
- 重置成员密码
- 调整成员角色
- 批量导入成员

管理员额外可以：

- 任命管理员
- 删除没有值班记录的成员

账号停用后：

- 不能登录后台。
- 不能提交签到或签退。
- 历史记录仍会保留。

如果成员已经有值班记录，建议停用账号，不建议删除。

## 8. 批量导入成员

会长和管理员可以在“成员”页面使用批量导入。

推荐流程：

1. 点击下载导入模板。
2. 按模板填写成员信息。
3. 上传 Excel 文件。
4. 查看导入结果。

模板字段包括：

| 字段 | 是否必填 | 说明 |
| --- | --- | --- |
| 学号 | 必填 | 用户唯一标识，不能重复 |
| 姓名 | 必填 | 成员姓名 |
| 手机号 | 可选 | 成员联系方式 |
| 学院 | 可选 | 成员学院 |
| 年级 | 可选 | 例如 2023 级 |
| QQ | 可选 | 成员 QQ |

导入规则：

- 新学号会创建为普通成员。
- 新成员初始密码默认为学号后六位。
- 已存在学号会更新姓名、手机号、学院、年级、QQ。
- 已存在成员的角色、账号状态和密码不会因为导入被修改。
- 文件内格式错误的行会被跳过，并在导入结果中显示原因。

## 9. 统计和导出

部长、会长、管理员可以进入“统计”页面查看统计数据。

统计支持：

- 自定义开始日期和结束日期
- 本周
- 本月
- 本学年
- 每人有效次数
- 每人有效时长合计
- 本周值班日明细

“本周值班日明细”的表格含义：

- 行：值班人员
- 列：本周每个值班日
- 数据：该成员当天的有效时长

会长和管理员可以导出 Excel。导出的表格格式为：

- 学号
- 姓名
- 按周分组的值班日列
- 有效总时长

同一周内的值班日会在第一行合并成一个周标题，例如“第一周（9月1日-9月5日）”。每个成员每个值班日对应的单元格显示当天有效时长，最后一列为当前查询范围内的有效总时长。

## 10. 培训管理

培训页面只对会长和管理员开放。部长和普通成员不能进入培训页面。

会长和管理员可以：

- 新增培训场次
- 编辑培训标题、日期、时间、地点、主讲人和说明
- 手动新增或修改参与记录
- Excel 导入参与名单
- 导出单场培训名单
- 导出时间范围内的培训统计
- 归档不再展示的培训

培训名单 Excel 推荐字段：

| 字段 | 是否必填 | 说明 |
| --- | --- | --- |
| 学号 | 必填 | 成员学号 |
| 姓名 | 必填 | 成员姓名；如果学号已在成员表中，系统会优先使用成员表姓名 |
| 时长 | 必填 | 计入值班统计的培训时长，单位为小时 |
| 备注 | 可选 | 例如补录原因、请假说明 |

导入规则：

- 同一场培训内，学号不能重复。
- 文件内重复学号会跳过。
- Excel 第一条数据默认作为主讲人记录。
- 已在该场培训中的学号会更新时长和备注。
- 成员表中不存在的学号也可以导入，会保存为外部快照记录。

## 11. 排班表

会长和管理员可以进入“排班”页面维护每周值班安排。

排班表支持：

- 按星期维护固定周排班
- 从“设置”页面已经保存的值班时间段中选择
- 设置标题、地点、备注
- 安排多个值班成员
- 控制某段排班是否显示在公开签到页
- 归档不再使用的排班

新增或编辑排班时，成员可以每行填写一个：

```text
2025000001 张三
李四
```

如果填写了学号且学号存在于成员表，系统会自动使用成员表中的姓名；如果只填写姓名，也会作为排班展示姓名保存。

公开签到页和后台首页会按设置中的值班时间段自动分组并同步显示排班。

## 12. 值班星期设置

会长和管理员可以进入“设置”页面调整值班星期和值班时间段。排班时只能选择已经保存的时间段。

例如设置为星期一、星期三、星期五后，只有这些日期允许公开页普通签到签退。

修改值班星期只影响之后的签到签退判断，不建议频繁改动。

## 13. 维修事务

部长、会长和管理员可以进入“维修”页面登记协会维修事务。

维修事务支持：

- 新增接修信息
- 记录送修人和联系方式
- 记录设备类型、品牌和型号
- 记录故障描述、处理记录和备注
- 状态只使用“进行中”“已完成”“已取消”
- 确认数据备份提醒、维修风险和隐私提示
- 选择并打印“维修协议”或“免责协议”
- 会长和管理员按日期范围导出维修事务 Excel；部长不能导出

新建事务时系统会自动生成编号，格式类似：

```text
WX202607070001
```

如果只是测试或录入错误，可以把状态改为“已取消”。默认“进行中”筛选不会显示已取消事务；需要查看时选择“全部状态”或“已取消”。

## 14. 数据中心

会长和管理员可以进入“数据”页面处理离线数据交接和归档。

数据中心支持：

- 查看成员档案、签到记录、培训数据、排班表、维修事务、操作日志和备份数量
- 下载成员批量导入模板
- 按日期范围导出值班统计 Excel
- 按日期范围导出培训统计 Excel
- 按日期范围导出维修事务 Excel
- 管理员导出操作日志 Excel
- 创建、下载最近备份
- 管理员上传系统生成的 `backup_*.zip` 恢复数据

数据中心分为“导入与导出”和“备份与恢复”两个区域，完整备份列表、下载、删除和恢复入口均集中在后者。

会长和管理员可以删除签到记录。删除前系统会自动创建安全备份；只有管理员可以修改签到时间、删除备份、恢复备份和查看操作日志。

## 15. 日常使用建议

每次正式值班前：

1. 确认网站可以正常打开。
2. 确认当天是值班日。
3. 确认需要值班的成员账号没有被停用。

值班过程中：

1. 成员到达时输入学号或姓名签到。
2. 离开时再次输入学号或姓名签退。
3. 如果显示姓名不对，不要继续提交，先联系会长或管理员检查账号。

值班结束后：

1. 部长、会长或管理员登录后台。
2. 进入“审核”页面处理待审核记录。
3. 确认明显异常的记录是否需要驳回。
4. 进入“统计”页面查看有效时长。

每周或每月：

1. 在“统计”页面选择时间范围。
2. 检查本周值班日明细和汇总数据。
3. 需要留档时由会长或管理员导出 Excel。
4. 建议定期运行数据库备份。

### 15.1 数据备份与恢复

会长和管理员可以在“数据”页面的“备份与恢复”区域执行一键备份，备份内容包括成员、培训数据、排班数据、维修事务、签到记录、操作日志、值班星期配置和系统配置。

只有管理员可以恢复备份。恢复时需要上传系统生成的 `backup_*.zip` 文件，系统会先自动备份当前数据，再覆盖恢复成员、培训数据、排班数据、维修事务、签到记录、操作日志、值班星期配置和系统配置。恢复成功后需要重新登录后台。

应用主数据库位于根目录 `data/attendance.db`。桌面窗口关闭后系统仍会在托盘运行；迁移电脑时必须先从托盘菜单选择“完全退出”，再复制包含 `app`、`data`、`backups`、`exports` 和 `logs` 的完整根目录。应用运行期间不要直接复制数据库文件。

## 16. 常见问题

### 16.1 输入学号或姓名后提示不存在

可能原因：

- 学号输入错误。
- 成员还没有被添加到系统。
- 批量导入时该行没有成功导入。

处理方式：联系会长或管理员检查成员列表。

### 16.2 提示账号已停用

该账号不能登录，也不能签到签退。

处理方式：联系会长或管理员确认是否需要重新启用。

### 16.3 忘记密码

普通成员、部长、会长可以联系会长或管理员重置密码。

会长或管理员可以在成员页面重置其他成员的密码。建议至少保留两个可用的管理员账号，并妥善保存管理员凭据；系统不提供绕过登录的本机密码恢复入口。

### 16.4 忘记签退

记录会保留为未签退状态，不会计入有效时长。

如确实需要补改，由管理员处理，并填写修改原因。

### 16.5 统计里没有有效时长

可能原因：

- 还没有签退。
- 签到或签退还没有审核通过。
- 签到或签退被驳回。
- 当天不是值班日。
- 实际时长不足 30 分钟，四舍五入后为 0 小时。

### 16.6 导入成员后没有看到角色变化

批量导入只维护基础资料，不会修改已有成员的角色、账号状态和密码。

需要调整角色时，请在“成员”页面单独修改。
