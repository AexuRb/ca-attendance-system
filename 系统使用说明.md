# 计算机协会值班签到签退系统使用说明

本文档面向日常使用人员，用来说明系统怎么打开、不同角色能做什么、每天如何签到签退、如何审核、如何管理成员以及如何统计导出。

开发、数据库和启动脚本相关内容请看 `本地运行说明.md`。

## 1. 打开系统

在运行网站的电脑上访问：

```text
http://127.0.0.1:8080
```

如果需要让同一局域网内的其他电脑访问，可以把 `127.0.0.1` 换成运行网站电脑的局域网 IP，例如：

```text
http://192.168.1.10:8080
```

打开后默认进入公开签到页。公开签到页不需要登录，输入学号或姓名即可提交签到或签退。

## 2. 角色权限

系统有四种角色：成员、部长、会长、管理员。

| 角色 | 主要用途 |
| --- | --- |
| 成员 | 日常值班签到签退，登录后查看自己的值班记录，维护个人资料和修改密码 |
| 部长 | 协助审核成员和其他部长的值班记录，查看统计数据 |
| 会长 | 管理成员、批量导入、调整角色、启用停用账号、调整值班星期、查看统计并导出 |
| 管理员 | 系统维护最高权限，额外可以任命管理员、删除无记录成员、查看维护日志 |

需要注意：

- 成员和部长提交的签到、签退需要审核。
- 会长和管理员提交的签到、签退自动通过。
- 部长不能审核自己的记录。
- 会长不能任命或维护管理员身份。
- 管理员只能由管理员任命。
- 成员管理功能只对会长和管理员开放。
- 删除成员只对管理员开放，并且只能删除没有值班记录的成员。

## 3. 日常签到签退

公开签到页操作流程：

1. 打开系统首页。
2. 输入学号或姓名。
3. 系统显示姓名，确认是本人。
4. 点击提交。

如果输入姓名后出现重名，系统会列出所有同名成员，请选择自己对应的学号，再继续确认签到或签退。

系统会自动判断本次是签到还是签退：

- 如果当前没有未签退记录，本次提交为签到。
- 如果当前已有未签退记录，本次提交为签退。

同一成员同一天允许多次签到签退。每一组“签到到签退”会形成一条值班记录。

如果页面提示“今日不是值班日”，说明当天没有被设置为值班星期，普通签到签退不会提交。

## 4. 审核值班记录

部长、会长、管理员登录后台后，可以进入“审核”页面处理待审核记录。

审核时可以：

- 通过签到
- 驳回签到
- 通过签退
- 驳回签退

驳回时需要填写原因。被驳回的记录不会删除，但不会计入有效统计。

一条记录只有在签到和签退都通过或自动通过后，才会计算有效时长。

## 5. 有效时长规则

有效时长等于签退时间减去签到时间，单位为小时，按分钟四舍五入取整数。

示例：

| 实际时长 | 有效时长 |
| --- | --- |
| 1 小时 15 分钟 | 1 小时 |
| 2 小时 34 分钟 | 3 小时 |
| 0 小时 25 分钟 | 0 小时 |

以下情况有效时长为 0：

- 只有签到，没有签退。
- 签到或签退被驳回。
- 签退时间早于签到时间。
- 记录不属于值班日。

## 6. 成员个人中心

所有角色登录后都可以进入“资料”页面。

可以修改：

- 年级
- 登录密码

年级为筛选项，范围是 2007 级到 2057 级。

成员也可以在“资料”页面查看自己的值班记录、审核状态和有效时长。

## 7. 成员管理

会长和管理员可以进入“成员”页面管理成员。

支持的操作：

- 新增成员
- 编辑成员信息
- 按关键词搜索成员
- 按年级筛选成员
- 按角色筛选成员
- 单个启用或停用账号
- 对筛选后的成员批量启用或批量停用
- 重置成员密码
- 调整成员角色
- 批量导入成员

管理员额外可以：

- 任命管理员
- 删除没有值班记录的成员

账号停用后：

- 不能登录后台。
- 不能提交签到或签退。
- 历史记录仍会保留。

如果成员已经有值班记录，建议停用账号，不建议删除。

## 8. 批量导入成员

会长和管理员可以在“成员”页面使用批量导入。

推荐流程：

1. 点击下载导入模板。
2. 按模板填写成员信息。
3. 上传 Excel 文件。
4. 查看导入结果。

模板字段包括：

| 字段 | 是否必填 | 说明 |
| --- | --- | --- |
| 学号 | 必填 | 用户唯一标识，不能重复 |
| 姓名 | 必填 | 成员姓名 |
| 年级 | 可选 | 例如 2023 级 |

导入规则：

- 新学号会创建为普通成员。
- 新成员初始密码默认为学号后六位。
- 已存在学号会更新姓名、年级。
- 已存在成员的角色、账号状态和密码不会因为导入被修改。
- 文件内格式错误的行会被跳过，并在导入结果中显示原因。

## 9. 统计和导出

部长、会长、管理员可以进入“统计”页面查看统计数据。

统计支持：

- 自定义开始日期和结束日期
- 本周
- 本月
- 本学年
- 每人有效次数
- 每人有效时长合计
- 本周值班日明细

“本周值班日明细”的表格含义：

- 行：值班人员
- 列：本周每个值班日
- 数据：该成员当天的有效时长

会长和管理员可以导出 Excel。导出的表格格式为：

- 学号
- 姓名
- 按周分组的值班日列
- 有效总时长

同一周内的值班日会在第一行合并成一个周标题，例如“第一周（9月1日-9月5日）”。每个成员每个值班日对应的单元格显示当天有效时长，最后一列为当前查询范围内的有效总时长。

## 10. 值班星期设置

会长和管理员可以进入“设置”页面调整值班星期。

例如设置为星期一、星期三、星期五后，只有这些日期允许公开页普通签到签退。

修改值班星期只影响之后的签到签退判断，不建议频繁改动。

## 11. 日常使用建议

每次正式值班前：

1. 确认网站可以正常打开。
2. 确认当天是值班日。
3. 确认需要值班的成员账号没有被停用。

值班过程中：

1. 成员到达时输入学号或姓名签到。
2. 离开时再次输入学号或姓名签退。
3. 如果显示姓名不对，不要继续提交，先联系会长或管理员检查账号。

值班结束后：

1. 部长、会长或管理员登录后台。
2. 进入“审核”页面处理待审核记录。
3. 确认明显异常的记录是否需要驳回。
4. 进入“统计”页面查看有效时长。

每周或每月：

1. 在“统计”页面选择时间范围。
2. 检查本周值班日明细和汇总数据。
3. 需要留档时由会长或管理员导出 Excel。
4. 建议定期运行数据库备份。

## 12. 常见问题

### 12.1 输入学号或姓名后提示不存在

可能原因：

- 学号输入错误。
- 成员还没有被添加到系统。
- 批量导入时该行没有成功导入。

处理方式：联系会长或管理员检查成员列表。

### 12.2 提示账号已停用

该账号不能登录，也不能签到签退。

处理方式：联系会长或管理员确认是否需要重新启用。

### 12.3 忘记密码

普通成员、部长、会长可以联系会长或管理员重置密码。

管理员账号由管理员维护。

### 12.4 忘记签退

记录会保留为未签退状态，不会计入有效时长。

如确实需要补改，由管理员处理，并填写修改原因。

### 12.5 统计里没有有效时长

可能原因：

- 还没有签退。
- 签到或签退还没有审核通过。
- 签到或签退被驳回。
- 当天不是值班日。
- 实际时长不足 30 分钟，四舍五入后为 0 小时。

### 12.6 导入成员后没有看到角色变化

批量导入只维护基础资料，不会修改已有成员的角色、账号状态和密码。

需要调整角色时，请在“成员”页面单独修改。
