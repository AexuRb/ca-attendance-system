# 本地运行说明

系统设计为单台 Windows 电脑上的本地离线服务，默认只允许本机通过 `127.0.0.1` 访问。

## 1. 环境要求

运行 Release 包需要：

- JDK 21
- MySQL 8.x

只有从源码重新构建时才需要 Node.js 和 Maven。

## 2. 配置本机参数

首次运行时，将：

```text
local-config.example.bat
```

复制为：

```text
local-config.bat
```

然后设置 MySQL 密码和初始管理员密码。默认参数为：

```text
APP_HOST=127.0.0.1
APP_PORT=8080
```

`local-config.bat` 包含本机密码，不会上传到 GitHub，也不应放入 Release 包。

如果直接运行 `start.bat` 时缺少该文件，脚本会创建模板并停止。修改模板后重新运行即可。

## 3. 初始化数据库

首次使用或更换电脑时，双击：

```text
init-db.bat
```

脚本会读取 `database/schema.sql` 和 `database/seed.sql`。如果找不到 MySQL 客户端，请把 MySQL 的 `bin` 目录加入 `PATH`，或在 `local-config.bat` 中配置 `MYSQL_CLIENT`。

## 4. 启动系统

双击项目根目录下的：

```text
start.bat
```

脚本优先运行 Release 包中已经构建的 JAR；如果 JAR 不存在，会在检测到 Maven 后尝试构建。启动成功后浏览器会打开：

```text
http://127.0.0.1:8080
```

如果提示端口已被占用，通常表示系统已经运行，可以直接访问上述地址；也可以在 `local-config.bat` 中修改 `APP_PORT`。

## 5. 登录与账号

在签到台右上角点击后台图标进入身份验证页。登录页可以记住账号，但不会保存密码。

初始管理员账号和密码由本机初始化配置决定。首次登录后应立即在个人中心修改密码。

## 6. 数据备份与恢复

会长或管理员可以进入“数据”或“系统 > 维护”执行一键备份。备份文件保存在：

```text
backups/app
```

备份内容包括成员、签到记录、培训、排班、维修事务、操作日志、值班星期、值班时段和系统配置。

只有管理员可以删除备份或上传系统生成的 `backup_*.zip` 恢复数据。恢复前系统会自动备份当前数据，恢复完成后需要重新登录。

## 7. 从源码重新构建

修改前端后执行：

```bash
cd frontend
npm install
npm run build
```

然后构建后端：

```bash
cd ../backend
mvn test
mvn package
```

生成的 JAR 位于：

```text
backend/target/attendance-backend-0.0.1-SNAPSHOT.jar
```

## 8. 常见问题

### 数据库提示 `using password: NO`

`local-config.bat` 没有正确设置 `DB_PASSWORD`，后端正在尝试使用空密码。

### 页面无法打开

确认 MySQL 已启动，并查看启动窗口中的后端错误信息。也可以运行：

```text
scripts\check-port.ps1
scripts\check-db.ps1
```

### 修改前端后页面没有变化

先执行 `npm run build`，再重新运行 `mvn package` 并重启服务。浏览器仍显示旧页面时可使用 `Ctrl + F5` 强制刷新。
