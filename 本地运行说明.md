# 本地运行说明

系统面向单台 Windows 电脑离线运行，只监听 `127.0.0.1:8080`，不会开放局域网端口。

## 1. 选择发行包

日常使用支持 Windows 10/11 x64，不需要安装 Java、MySQL、Node.js 或 Maven，也不需要联网。

### 安装版

运行 `CA-Attendance-System-Setup-2.0.0.exe`，按向导安装。默认目录为：

```text
C:\CAAttendance\app
```

开始菜单和桌面快捷方式会直接启动应用。业务数据保存在 `C:\CAAttendance` 下，不放进 `app` 程序目录。

### 便携版

解压 `CA-Attendance-System-Portable-2.0.0.zip`，保持目录结构不变，双击：

```text
启动管理系统.bat
```

也可以直接运行 `app\CA-Attendance-System.exe`。便携版适合整文件夹复制到另一台电脑。

## 2. 首次初始化

首次启动时，程序会自动创建：

```text
data\attendance.db
```

随后显示“系统首次初始化”，由现场人员填写管理员账号、姓名和密码。系统没有预置公开管理员密码，也不需要执行数据库脚本。

初始化完成后应立即进入“系统 > 设置”配置值班星期和值班时间段，再进入“值班 > 排班”安排部长。

## 3. 启动与退出

桌面应用启动后会自动拉起本机后端并打开签到台。应用运行期间，可同时在本机浏览器访问：

```text
http://127.0.0.1:8080
```

点击桌面窗口右上角关闭按钮后，窗口会隐藏到 Windows 托盘，本机后端和浏览器访问仍会继续运行。点击托盘图标可以重新打开窗口；需要停止服务时，在托盘菜单中选择“完全退出”，程序会先请求后端正常停机，再退出内置 Java。

复制数据、更新程序或关机前，应通过托盘菜单“完全退出”，并确认本机服务已经停止。

## 4. 目录结构

```text
CAAttendance\
├─ app\                     可替换的程序文件
├─ data\attendance.db       SQLite 主数据库
├─ backups\app\             后台一键备份
├─ exports\                 导出目录
└─ logs\                    desktop.log 与 backend.log
```

更新程序时只替换 `app`；不要删除或覆盖 `data`、`backups`、`exports` 和 `logs`。数据库和备份未加密，请限制无关人员访问该文件夹。

## 5. 备份、恢复与迁移

会长或管理员可以在“数据”或“系统 > 维护”执行一键备份，文件保存在：

```text
backups\app
```

备份包含成员、签到、培训、排班、维修、日志和值班设置。只有管理员可以恢复或删除备份；恢复前系统会自动保存当前数据，完成后所有账号需要重新登录。

整机迁移步骤：

1. 从托盘菜单选择“完全退出”，确认 `127.0.0.1:8080` 已无法访问。
2. 复制整个 `CAAttendance` 文件夹到新电脑。
3. 从新目录启动 `app\CA-Attendance-System.exe`。
4. 登录后台检查成员、排班和最近备份。

不要在应用运行时直接复制 `attendance.db`；运行中的数据请通过后台备份导出。

## 6. 忘记密码

成员密码由会长或管理员在成员页面重置。建议至少保留两个可用的管理员账号，并妥善保存管理员凭据；系统不提供绕过登录的本机密码恢复入口。

## 7. 未签名安装包

当前项目没有代码签名证书。Windows 首次运行时可能显示“未知发布者”或 Microsoft Defender SmartScreen 信誉提示，这是未签名程序的正常现象。

应从项目正式 Release 获取文件，并使用同一 Release 中的 `SHA256SUMS.txt` 校验：

```powershell
Get-FileHash .\CA-Attendance-System-Setup-2.0.0.exe -Algorithm SHA256
```

结果应与校验文件中对应行完全一致。不要关闭系统安全功能，也不要运行来源不明的同名安装包。

## 8. 源码运行

源码开发需要：

- JDK 21
- Maven 3.9+
- Node.js 22.12+

不需要安装 MySQL。修改前端后执行：

```powershell
cd frontend
npm ci
npm run build

cd ..\backend
mvn package
```

生成的后端文件为：

```text
backend\target\attendance-backend.jar
```

随后双击根目录 `start.bat`，或在 PowerShell 中启动后端。源码版使用电脑上已安装的 JDK；业务数据仍默认写入项目根目录。

## 9. 构建 Windows 发行包

在仓库根目录运行：

```powershell
.\scripts\build-desktop.ps1
```

脚本会：

1. 下载并校验固定版本 Eclipse Temurin JRE 21。
2. 使用干净依赖构建 Vue 前端。
3. 运行 Maven 测试并打包后端。
4. 运行桌面端测试并构建 NSIS 安装程序。
5. 生成便携 ZIP 和 `SHA256SUMS.txt`。

最终文件位于 `release-artifacts\`。构建过程首次需要联网下载依赖，生成的发行包运行时不需要联网。

## 10. 常见问题

### 8080 端口已被占用

先确认是否已经启动过本系统。若是其他程序占用，请关闭该程序后重试。桌面版固定使用 8080，以保证浏览器访问地址一致。

### 页面无法打开

查看根目录 `logs\desktop.log` 和 `logs\backend.log`。若数据库完整性检查失败，应保留当前 `data` 文件夹并从后台备份恢复，不要直接编辑 SQLite 文件。

### 修改前端后页面没有变化

先运行 `npm run build`，再运行 `mvn package` 并重启服务。浏览器仍缓存旧页面时可使用 `Ctrl + F5`。

### 首次启动没有出现初始化页

说明当前根目录已经存在用户数据。确认程序实际所在根目录，并检查 `data\attendance.db`；不要为了重新初始化而随意删除真实数据库。
